# ROADMAP.md

## Phase 1: Project Foundation

- Create operating documentation.
- Convert profile templates into Markdown working templates.
- Define safety rules and review expectations.
- Capture durable decisions.

## Phase 2: Evidence and Data Model

- Define the internal claim object.
- Define source quality and confidence levels.
- Decide how structured notes should be represented.
- Decide what data may and may not appear in final outputs.

## Phase 3: First Skills

Suggested order:

1. Source Quality Checker.
2. Individual Prospect Profile Builder.
3. Organisation Prospect Profile Builder.
4. Due Diligence Screen.
5. Event Brief Builder.

## Phase 4: Prototype Workflow

- Accept user-provided source material.
- Convert source material into structured claims.
- Review claims for confidence, source quality, sensitivity, and UC connection support.
- Draft a short or standard profile.
- Run a due diligence and quality pass.
- Produce a polished final profile.

## Phase 5: Automation and Tooling

- Decide whether the workflow should remain Codex-native or become a small local app.
- Add tests and fixtures for profile safety rules.
- Add repeatable command-line or UI entry points if useful.
- Consider export formats after the workflow is stable.
