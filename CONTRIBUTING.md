# CONTRIBUTING.md

## Contributing

This project should evolve through small, reviewable changes. Keep documentation, templates, and future implementation aligned.

## Ground Rules

- Read `AGENTS.md` before making changes.
- Keep profile outputs conservative and evidence-led.
- Preserve separate treatment for individual and organisation profiles.
- Update `STATUS.md` after meaningful work.
- Update `DECISIONS.md` when a project choice should persist.
- Update `CHANGELOG.md` for structural or user-visible changes.

## Review Focus

Reviewers should prioritise:

- unsupported claims,
- invented UC connections,
- private or donor-level data exposure,
- overconfident wealth or philanthropy language,
- empty or filler final-profile sections,
- missing updates to docs after behaviour changes.
